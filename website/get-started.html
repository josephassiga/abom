Get started

Scan your first agent in two minutes.

Install ABOM, then produce a signed bill of materials from local code, a GitHub repo, or a live MCP endpoint — and block an unauthorized action with the gate.

$pip install abom-cli
1

Install

ABOM is an open-source CLI on PyPI. Requires Python 3.10+.

shell
pip install abom-cli          # the scanner, gate & verifier
pip install 'abom-cli[mcp]'   # add this to scan live MCP endpoints

abom version                  # → abom 0.1.x · ABOM spec v0.1
Tip: if you installed with pipx, add the MCP support with pipx inject abom-cli httpx. See troubleshooting if a command can't find a module.
2

Scan a local agent what it's made of

Point abom scan at any agent project. It detects the models, tools, prompts, data sources, frameworks and MCP servers it's built from — each with the exact DETECTED FROM reference — and emits a signed manifest.

shell
abom scan .                        # → abom.json (ed25519-signed)

  ABOM · my-agent @ 1.0.0  —  6 components
  ┌───┬───────────┬───────────────┬──────────┬──────────────────────┐
  │ # │ TYPE      │ NAME          │ VERSION  │ DETECTED FROM        │
  ├───┼───────────┼───────────────┼──────────┼──────────────────────┤
  │ 1 │ model     │ OpenAI (SDK)  │ 1.51.0   │ dependency:openai    │
  │ 2 │ framework │ LangChain     │ >=0.2.16 │ dependency:langchain │
  │ 3 │ tool      │ lookup_order  │ —        │ file:agent.py:14     │
  └───┴───────────┴───────────────┴──────────┴──────────────────────┘
  signed: ed25519 · key 7f7aa8d6…

Verify the signature any time — it's self-contained, so anyone you hand the file to can check it:

shell
abom verify abom.json              # → ✓ VALID — signature OK

Block an unauthorized action what it's allowed to do

The gate enforces the signed manifest at the tool boundary, deny-by-default. A tool that isn't declared is blocked before it runs — not logged after.

shell · CLI
abom gate abom.json --tool lookup_order    # → ALLOW (exit 0)
abom gate abom.json --tool wire_transfer   # → DENY  (exit 1)
                                           #   tool_not_in_manifest

Or wrap a tool in your agent code — a prompt injection that calls an undeclared tool raises ActionDenied before the body executes:

agent.py
from abom import Gate, ActionDenied

gate = Gate(signed_manifest)

@gate.gated()
def wire_transfer(amount, to):
    ...                       # never runs if undeclared

try:
    wire_transfer(1_000_000, to="attacker")
except ActionDenied as e:
    print(e.decision.rule)    # → "tool_not_in_manifest"
    # the money never moved — and the denial is notarized

Scan a remote agent three ways

You don't need the code locally. ABOM scans a GitHub repo by URL, or a live deployed agent over the Model Context Protocol.

scan <path>

Local code

A directory or repo on your machine.

scan --url

A GitHub repo

Clones, scans, and cleans up. Never runs the code.

scan --mcp

A live endpoint

Enumerates a running agent's declared capabilities.

shell · a public repo
abom scan --url https://github.com/assafelovic/gpt-researcher

  ABOM · gpt-researcher @ 0.14.7  —  25 components
  # models, frameworks, tools, MCP server — versioned, signed
shell · a live MCP server
abom scan --mcp https://mcp.deepwiki.com/mcp

  ABOM · DeepWiki @ 2.14.3  —  4 components
  │ tool       │ ask_question        │ mcp:https://mcp.deepwiki.com/mcp │
  │ tool       │ read_wiki_contents  │ mcp:https://mcp.deepwiki.com/mcp │
  │ MCP server │ DeepWiki            │ mcp:https://mcp.deepwiki.com/mcp │
Auth-gated servers (like GitHub's https://api.githubcopilot.com/mcp/) work too — pass a token with --token <PAT> or the ABOM_MCP_TOKEN env var. ABOM only reads the declared capabilities; it never invokes a tool.
5

Make it shareable

Add --card to any scan or gate to also write a self-contained SVG "report card" you can drop into a PR, a doc, or a slide.

shell
abom scan . --card abom-card.svg
abom gate abom.json --tool wire_transfer --card decision.svg

And it drops into CI — abom gate exits non-zero on DENY, so an unauthorized action fails the build:

.github/workflows/ci.yml
- uses: josephassiga/abom-dev/.github/actions/abom-scan@main
  with:
    path: .

Troubleshooting

A couple of real-world Python gotchas, and how to fix them.

SymptomFix
The abom command says No module named httpx even though you installed it Your abom binary (often a pipx install) uses a different environment than your pip. Either pipx inject abom-cli httpx, or run it via the right interpreter: python -m abom scan --mcp …
import abom reports an old version A stale copy is shadowing the install. Reinstall cleanly: pip install -U abom-cli (or pip install -e . from a source checkout for the latest).
--no-banner "no such option" It's a top-level flag — put it before the command: abom --no-banner scan …
Piping scan -o - into jq Works cleanly — the banner and summary go to stderr, so stdout is pure JSON.
