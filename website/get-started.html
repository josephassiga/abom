Block an unauthorized action what it's allowed to do

The gate enforces the signed manifest at the tool boundary, deny-by-default. A tool that isn't declared is blocked before it runs вҖ” not logged after.

Start from a manifest that declares which tools the agent may use:

gate.json
{
  "abom": "0.1",
  "type": "CompositionManifest",
  "agent": { "name": "support-agent", "version": "1.0.0" },
  "components": [
    { "type": "tool", "name": "lookup_order" },
    { "type": "tool", "name": "http_fetch", "scope": "egress",
      "allowed_endpoints": ["api.internal.shop"] }
  ]
}

A declared tool is allowed (exit 0):

Or wrap a tool in your agent code вҖ” a prompt injection that calls an undeclared tool raises ActionDenied before the body executes:

agent.py
from abom import Gate, ActionDenied

gate = Gate(signed_manifest)

@gate.gated()
def wire_transfer(amount, to):
    ...                       # never runs if undeclared

try:
    wire_transfer(1_000_000, to="attacker")
except ActionDenied as e:
    print(e.decision.rule)    # вҶ’ "tool_not_in_manifest"
    # the money never moved вҖ” and the denial is notarized

Or move the gate out of the agent's process entirely with the MCP broker. The agent connects to the broker instead of its MCP server; every tools/call is checked against the signed manifest before it's forwarded, so a call the agent was never granted never reaches the server:

shell В· the MCP broker
abom broker abom.json --upstream https://mcp.internal/rpc \
             --trusted-key 9a72f7efвҖҰ --log gate.mlog

Honest scope: enforcement is structural for brokered tools. It doesn't stop an agent from making raw out-of-band calls (HTTP, subprocess) that never transit the broker вҖ” pair it with network egress containment for that.

Prove the record can't be rewritten verify without trusting the operator

Every gate decision lands in an append-only Merkle transparency log вҖ” the same RFC 6962 construction behind Certificate Transparency and Sigstore. Two independent steps make that record something a third party can rely on, even against the operator who runs it.

First, run the gate or broker with --log so decisions land in a durable transparency log:

shell
abom gate abom.json --tool lookup_customer --log gate.mlog
abom broker abom.json --upstream https://mcp.internal/rpc --log gate.mlog

A witness cosigns the log with its own key вҖ” separate from the operator's. It refuses to cosign unless the log only grew since it last saw it, so a rewrite or truncation is rejected:

shell В· the witness
abom witness gate.mlog
  вң“ cosigned  size=2  root 1ffb0c515b588b7dвҖҰ
  witness key 9987a9c4c7d5f2c8 В· вҶ’ gate.mlog.cosig.json

# operator rewrites the log and tries again:
abom witness gate.mlog
  вң— WITNESS REFUSED вҖ” a log may only grow (append-only)

Anchor the root to an RFC 3161 timestamp authority вҖ” an independent third party the operator doesn't control. Under eIDAS Art. 41 a qualified timestamp carries a legal presumption of time and integrity, proving the root existed, unaltered, by time T:

shell В· the anchor
abom anchor gate.mlog --tsa https://freetsa.org/tsr
  вң“ anchored  root 1ffb0c515b588b7dвҖҰ
  RFC 3161 token В· вҶ’ gate.mlog.anchor.json  # re-verifiable offline, later

# the auditor's check вҖ” no trust in the operator:
abom witness-verify gate.mlog.cosig.json
  вң“ VALID вҖ” witness attests size=2, root 1ffb0c51вҖҰ
Honest scope: a witness you host yourself is defense-in-depth вҖ” the operator holds both keys. Genuine non-equivocation comes from an independent witness (the neutral network is the next step). The RFC 3161 timestamp is the third-party proof that stands today.

Scan a remote agent three ways

You don't need the code locally. ABOM scans a GitHub repo by URL, or a live deployed agent over the Model Context Protocol.

scan --url

A GitHub repo

Clones, scans, and cleans up. Never runs the code.

scan --mcp

A live endpoint

Enumerates a running agent's declared capabilities.

Add --card to any scan or gate to also write a self-contained SVG "report card" you can drop into a PR, a doc, or a slide.

shell
abom scan . --card abom-card.svg
abom gate abom.json --tool wire_transfer --card decision.svg

And it drops into CI вҖ” abom gate exits non-zero on DENY, so an unauthorized action fails the build:

.github/workflows/ci.yml
- uses: josephassiga/abom-dev/.github/actions/abom-scan@main
  with:
    path: .

Troubleshooting

A couple of real-world Python gotchas, and how to fix them.

SymptomFix
The abom command says No module named httpx even though you installed it Your abom binary (often a pipx install) uses a different environment than your pip. Either pipx inject abom-cli httpx, or run it via the right interpreter: python -m abom scan --mcp вҖҰ
import abom reports an old version A stale copy is shadowing the install. Reinstall cleanly: pip install -U abom-cli (or pip install -e . from a source checkout for the latest).
--no-banner "no such option" It's a top-level flag вҖ” put it before the command: abom --no-banner scan вҖҰ
Piping scan -o - into jq Works cleanly вҖ” the banner and summary go to stderr, so stdout is pure JSON.
